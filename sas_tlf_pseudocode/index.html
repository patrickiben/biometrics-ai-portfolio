Before you start

You'll need: SAS 9.4+ (or SAS Studio) on a machine that can reach your study's CDISC ADaM datasets  ·  the relevant ADaM tables for the TLF you're running — at minimum ADSL, plus the domain the program reads (ADAE for AE tables, ADLB for labs, ADVS/ADEG for vitals/ECG, ADPC/ADPP for PK, ADPD for PD, ADIS for immunogenicity, ADEX for exposure)  ·  write access to an output folder for the result datasets  ·  the study's format catalog (treatment order, AE severity, lab ranges) referenced by %setup. No internet, no install beyond SAS itself. These are pseudocode skeletons — open one, point a LIBNAME at your ADaM, and complete the spec; don't expect a clean run out of the box.

How to run a program

There's no single command-line driver — each .sas is one self-contained TLF program. The pattern is the same for all 217: open it in SAS → point a LIBNAME at your ADaM → submit the program. Each program already %includes the shared macros and calls %setup / %designvars for you, so the two things you edit are the ADaM path and the output path in the %setup call near the top.

Quick-start checklist — click each as you finish it:

  • Open the program for the TLF you want, in the folder for your study's design (e.g. parallel/t_demographics.sas)
  • Edit the %setup(...) line: set adam= to your ADaM library path and out= to your output folder
  • Confirm %designvars(design=...) matches the folder (e.g. PARALLEL for parallel/)
  • Make sure your study format catalog is available to %setup (treatment order, severity, lab ranges)
  • Submit the program in SAS; check the log for %include/%setup/%designvars resolving cleanly
  • Open the output dataset / TLF and reconcile it against the validated (double-programmed) result per SOP

Run it

Either edit the %setup line inside the program and submit it directly, or drive it from a tiny wrapper that sets the libname and %includes the program. Both do the same thing:

/* --- run one TLF program from a wrapper (SAS 9.4+) --------------- */
libname adam "/data/cp101/adam" access=readonly;   /* your ADaM datasets   */
libname tfl  "/data/cp101/tfl";                     /* your output folder   */

/* submit a program from the folder for your study's design:               *
 *   parallel/  crossover/  single_sequence/  sad/  mad/                    */
%include "parallel/t_demographics.sas";   /* one program -> one output      */

/* — or, equivalently, open parallel/t_demographics.sas in SAS, edit its    *
 *   %setup(study=CP-101, adam=/data/cp101/adam, out=/data/cp101/tfl);       *
 *   line to your paths, and just submit the whole program. —               */
Each program's own header shows the canonical call sequence: %include "../00_setup_macros.sas"; then %setup(study=, adam=, out=); then %designvars(design=<folder>); then the TLF body. Run from the design folder (or adjust the relative ../ path to 00_setup_macros.sas). Reported numbers always come from validated ADaM + double-programmed code per SOP — never from this scaffold or an LLM.

FolderDesignKey analysis differences
parallel/Parallel-groupBetween-group; treatment = TRT01A/TRT01AN
crossover/Crossover (2×2, Williams)Within-participant; by APERIOD/TRTSEQP; BE/ANOVA (mixed model, GMR + 90% CI)
single_sequence/Single-/fixed-sequence (e.g. DDI)By APERIOD; fixed order, no randomized sequence; ratio + 90% CI vs reference period

Design is also wired in code via %designvars(design=...) in 00_setup_macros.sas, so each design folder's programs differ only where the statistics genuinely differ (PK comparison, by-period/by-sequence safety, dose escalation) — Tables/Listings of safety are structurally identical, re-pointed to the design's treatment/period variables.

Naming convention

t_ table · l_ listing · f_ figure. One program → one output. Each program %includes 00_setup_macros.sas, calls %setup then %designvars(design=<folder>), then the TLF body.

ADaM variables relied on (no re-derivation here)

  • Treatment: TRT01A/TRT01AN (parallel), TRTA/TRTAN+APERIOD/APERIODC+TRTSEQP (crossover/seq)
  • Populations: SAFFL, PKFL, ITTFL, RANDFL
  • Analysis: PARAM/PARAMCD, AVAL, CHG, PCHG, BASE, AVISIT/AVISITN, ATPT/ATPTN, ANRIND, BNRIND
  • AE: TRTEMFL, AESOC, AEDECOD, ASEV/AESEVN, AREL, AESER, AESDTH, AEACN
  • PK: ADPC AVAL=conc, NRRELTM/ATPTN=nominal time; ADPP PARAMCD=Cmax/Tmax/AUClast/AUCinf/CL/Vz/t½/Rac
Geometric PK statistics are computed on the log scale (never exp(arithmetic mean)). AE counts are distinct participants. Single-sequence participant-level tables key on &SEQVAR/&SEQVARN while period tables denominate from a period-bearing source. The library is free of eDISH / Hy's Law terminology.

Troubleshooting

SymptomWhyFix
ERROR: Libref ADAM is not assigned / File ADAM.ADSL does not existThe LIBNAME in %setup doesn't point at your ADaM datasetsSet adam= in the %setup(...) call (or the wrapper libname adam) to the real folder holding ADSL/ADAE/…; it's opened access=readonly.
ERROR: Physical file does not exist … 00_setup_macros.sasThe program's %include "../00_setup_macros.sas" path is relative to where you submit fromSubmit from inside the design folder, or change the %include to the absolute path of 00_setup_macros.sas.
Columns/by-structure look wrong (e.g. no period split on a crossover)%designvars(design=...) doesn't match the folder you openedSet it to the folder's design: PARALLEL, CROSSOVER, SINGLESEQ, SAD, or MAD. It resolves &TRTVAR/&SEQVAR/&BYPERIOD.
ERROR: Format … not found in PROC REPORT/FORMATThe study format catalog (treatment order, AE severity, lab ranges) isn't available to %setupBuild/point tfl.formats at your study format catalog (the proc format library=tfl.formats step in 00_setup_macros.sas), then resubmit.
Geometric mean / Geo-CV% looks off vs the validated outputExpecting exp(arithmetic mean) — this library computes geo stats on the log scaleThat's intended: geo mean = exp(mean of logs), Geo-CV% = 100*sqrt(exp(var of logs)-1) (see %pkstats). Reconcile against the double-programmed value, not exp(mean).
AE counts higher than expectedTreating event rows as the unit instead of distinct participantsThe AE macros count count(distinct USUBJID) per level (treatment-emergent via TRTEMFL='Y'). Reconcile participant counts, not row counts.
Period table denominators don't add up (crossover / single-sequence)Denominating a per-period table from ADSL (one record per participant)Per-period denominators must come from a period-bearing source (ADEX/ADIS/ADPD); participant-level tables key on &SEQVAR/&SEQVARN.
Looking for an eDISH / Hy's-Law panelThe library deliberately omits that terminologyUse f_lab_lft_scatter.sas + the lab tables for transaminase/bilirubin review; there is no eDISH/Hy's Law output by design.
